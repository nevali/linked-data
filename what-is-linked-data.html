What is Linked Data?

Much of the World Wide Web today is made up of linked pages: documents, usually written in HTML, which are uniquely identifiable, so that part of one document can link to another, even if the link is the only thing those two documents have in common.

Linked Data applies the principles and mechanics used to connect documents intended for human consumption together on the Web to data—which is intended for software consumption.

In order to make this possible, Linked Data is built upon a set of principles (most of which should be familiar ground for Web developers who have never even so much as dabbled with Linked Data):

  1. the unit of currency—the means by which everything is identified—is the URI;
  2. things work best when we’re all speaking the same language; and
  3. use the facilities HTTP provides for us.

Over the following short chapters, I’m going to take you through each of these principles in turn (and even if you decide not to publish Linked Data, you’ll find that they’re a good set of guidelines to work from for publishing on the Web in general).
